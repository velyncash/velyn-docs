# Core Features

This document describes the **core features of Velyn.Cash** at a high level.
The goal is to explain **what the app does** and **why it exists**, without technical overload.

---

## 1. Account & Access

Velyn.Cash uses a **simple but secure access model**.

### What users get
- Username & password login
- PIN-based confirmation for sensitive actions
- Clear session-based access (no hidden background actions)

### Why it matters
Users can access their account quickly, while still having an extra security layer
without relying on complex flows.

---

## 2. Payments

Payments are the heart of Velyn.Cash.

### Supported payment methods
- **QR Payments**  
  Scan and pay instantly for everyday transactions.

- **NFC (Contactless Payments)**  
  Tap-to-pay using supported mobile devices.

- **Card Transactions**  
  Virtual and physical card usage for online and offline payments.

### Design focus
- Fast execution
- Clear confirmation
- Simple transaction history

---

## 3. Cards

Velyn provides **card-level control**, not just balances.

### Card features
- Virtual cards
- Physical cards (planned)
- Enable / disable cards
- Spending control (planned)

### Why this exists
Users should control how and where their money is used,
not rely on hidden bank rules.

---

## 4. Auto Budgeting

Velyn includes **automatic budgeting tools** to help users understand spending.

### Key capabilities
- Spending categorization
- Monthly summaries
- Simple insights (not complex finance charts)

### Goal
Help users stay aware of their money without needing financial knowledge.

---

## 6. Security by Design

Security is built into every feature.

### Core principles
- Encrypted sessions
- PIN confirmation
- No sensitive data exposed in UI
- Clear separation between UI and backend logic

Security is part of the system, not an optional add-on.

---

## 7. Transparency

Velyn is developed **in public**.

### What this means
- Active GitHub repositories
- Visible progress and commits
- Clear development status

This approach builds trust by showing real work, not promises.

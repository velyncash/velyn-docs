# Core Features

This document summarizes the **core product capabilities** of Velyn.Cash in a user-friendly way.

---

## 1) Account & Access

Velyn.Cash uses a simple access model that is still secure:
- **Username + Password**
- **PIN** for confirmation and secure access

Why it matters:
- Fast onboarding
- Familiar login
- Extra protection without complex flows

---

## 2) Payments

Payments are the primary use case.

Supported payment experiences:
- **QR Payments** — scan and pay in seconds
- **Contactless (NFC)** — tap-to-pay on supported devices
- **Card-based transactions** — virtual/physical card support (physical planned)
- **RFID scenarios** — controlled environments and fast tap flows (planned)

Design goals:
- Clear confirmation before finalizing
- Instant status feedback (success/failed/pending)
- Consistent transaction history

---

## 3) Cards

Cards are treated as **controlled payment instruments**, not passive balances.

Core capabilities:
- Virtual cards (initial focus)
- Physical cards (planned)
- Card controls (lock/unlock, visibility, limits planned)

---

## 4) Auto Budgeting

Budgeting is designed for clarity, not guilt.

Capabilities:
- Automatic spending categories
- Monthly spending overview
- Simple insights users can act on

---

## 5) Analytics & Insights

Analytics are lightweight and readable:
- Spending overview (period-based)
- Category breakdown
- Trends (daily/weekly/monthly)

No complicated finance dashboards.

---

## 6) Security by Design

Security is integrated into user flows:
- PIN confirmation when needed
- Session validation
- Minimal sensitive exposure in UI

---

## 7) Transparency

Velyn.Cash is developed publicly:
- Progress is visible via commits and repo updates
- Documentation is maintained as the source of truth
